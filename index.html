🌐 Arquitectura de Red: Protocolo FTP

Documentación técnica sobre el flujo de comunicación en FTP Activo y Pasivo.

🔴 1. FTP en Modo Activo (Active Mode)

Definición: El cliente inicia el canal de control, pero el servidor es quien inicia el canal de datos hacia el cliente.

Flujo de Comunicación

  1. El Cliente abre un puerto aleatorio N (>1023) y se conecta al puerto 21 del Servidor (Canal de Control).
  2. El Cliente envía el comando PORT N+1 informando dónde está escuchando.
  3. El Servidor usa su puerto local 20 para conectarse al puerto N+1 del cliente (Canal de Datos).

Diagrama de Secuencia

[Cliente] (Puerto N)   --- TCP SYN (Control) ---> [Servidor] (Puerto 21)
[Cliente] (Puerto N)   <-- TCP ACK (Control) ---  [Servidor] (Puerto 21)
[Cliente] (Puerto N)   --- Comando PORT N+1 --->  [Servidor] (Puerto 21)
[Cliente] (Puerto N+1) <-- TCP SYN (Datos) -----  [Servidor] (Puerto 20)
[Cliente] (Puerto N+1) --- TCP ACK (Datos) ---->  [Servidor] (Puerto 20)

Problema con Firewalls/NAT: Como el servidor inicia una conexión entrante hacia el cliente, los firewalls locales o routers NAT del cliente suelen bloquear esta solicitud, haciendo que falle la transferencia.

🟢 2. FTP en Modo Pasivo (Passive Mode o PASV)

Definición: El cliente inicia ambos canales (control y datos). Creado para sortear los bloqueos de Firewall del lado del cliente.

Flujo de Comunicación

  1. El Cliente se conecta al puerto 21 del Servidor (Canal de Control).
  2. El Cliente envía el comando PASV.
  3. El Servidor abre un puerto aleatorio P (>1023) y le dice al cliente: "Estoy escuchando pasivamente en el puerto P".
  4. El Cliente se conecta al puerto P del servidor para el Canal de Datos.

Diagrama de Secuencia

[Cliente] (Puerto N)   --- TCP SYN (Control) ---> [Servidor] (Puerto 21)
[Cliente] (Puerto N)   <-- TCP ACK (Control) ---  [Servidor] (Puerto 21)
[Cliente] (Puerto N)   --- Comando PASV ------->  [Servidor] (Puerto 21)
[Cliente] (Puerto N)   <-- Respuesta Puerto P --  [Servidor] (Puerto 21)
[Cliente] (Puerto N+1) --- TCP SYN (Datos) ---->  [Servidor] (Puerto P)
[Cliente] (Puerto N+1) <-- TCP ACK (Datos) -----  [Servidor] (Puerto P)

Problema con Firewalls/NAT: Traslada la carga administrativa al servidor. El servidor debe tener un rango de puertos altos abiertos en su Firewall para permitir las conexiones entrantes de los clientes.

💡 Nota del Arquitecto: El Modo Pasivo es el estándar de facto actual por la prevalencia de NATs en clientes. Sin embargo, para entornos modernos, se recomienda encarecidamente usar SFTP (SSH) sobre el puerto 22.

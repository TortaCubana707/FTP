🌐 Arquitectura de Red: El Protocolo FTP

Desde los conceptos más básicos hasta el flujo de comunicación técnico en FTP Activo y Pasivo.

📚 1. ¿Qué es FTP y por qué es "Especial"?

FTP (File Transfer Protocol) es uno de los protocolos más antiguos de Internet (creado en 1971). Sirve para transferir archivos entre un Cliente y un Servidor. Pero a diferencia de otros protocolos como HTTP (web) que usan una sola conexión para todo, FTP es especial porque usa dos conexiones simultáneas (dos canales) para funcionar.

👶 La Analogía del Bebé (El Sistema de Dos Teléfonos)

Imagina que quieres pedirle a una tienda que te envíe una caja de juguetes:

  • El Canal de Control (Puerto 21): Es como hacer una llamada telefónica a la tienda para decir: "Hola, soy yo. Mi contraseña es esta. Quiero la caja de juguetes número 5." Por aquí solo pasan órdenes y respuestas, no juguetes.
  • El Canal de Datos (Puerto 20 o Aleatorio): Es el camión de reparto que la tienda envía (o que tú mandas recoger) exclusivamente para llevar la caja de juguetes. Por aquí solo pasan los archivos, no se habla.

🔴 2. FTP en Modo Activo (Active Mode)

Definición: En el modo activo, el cliente inicia el "Canal de Control" (la llamada), pero es el servidor quien inicia la conexión del "Canal de Datos" (envía el camión) hacia el cliente.

Flujo de Comunicación Técnico

  1. El Cliente abre un puerto aleatorio N (ej. 5000) y se conecta al puerto 21 del Servidor (Canal de Control).
  2. El Cliente empieza a escuchar en el puerto siguiente N+1 (ej. 5001) y le envía al servidor el comando PORT 5001 ("Manda el camión a mi puerta 5001").
  3. El Servidor usa su puerto local 20 para iniciar una conexión hacia el puerto 5001 del cliente (Canal de Datos).

Diagrama de Secuencia TCP

[Cliente] (Puerto 5000)   --- TCP SYN (Control) ---> [Servidor] (Puerto 21)
[Cliente] (Puerto 5000)   <-- TCP ACK (Control) ---  [Servidor] (Puerto 21)
[Cliente] (Puerto 5000)   --- Comando PORT 5001 ---> [Servidor] (Puerto 21)
[Cliente] (Puerto 5001)   <-- TCP SYN (Datos) -----  [Servidor] (Puerto 20)
[Cliente] (Puerto 5001)   --- TCP ACK (Datos) ---->  [Servidor] (Puerto 20)
⚠️ El Problema del Modo Activo: El servidor intenta "entrar" a la red del cliente de forma no solicitada desde la perspectiva del router. Los routers caseros (NAT) y los Firewalls bloquean al servidor porque piensan: "Alguien de afuera está intentando conectarse a mi puerto 5001 sin que yo lo haya pedido internamente. ¡Bloqueado!".

🟢 3. FTP en Modo Pasivo (Passive Mode o PASV)

Definición: Para solucionar el problema de los firewalls del cliente, se inventó el Modo Pasivo. Aquí, el cliente inicia AMBAS conexiones (tanto el Canal de Control como el de Datos). El servidor simplemente se queda "pasivo" esperando a que el cliente se conecte para descargar los datos.

Flujo de Comunicación Técnico

  1. El Cliente se conecta al puerto 21 del Servidor (Canal de Control).
  2. El Cliente envía el comando PASV ("Dime dónde recojo el paquete").
  3. El Servidor abre un puerto aleatorio P (ej. 6000) y le responde: "Estoy escuchando pasivamente en mi puerto 6000".
  4. El Cliente abre otro puerto N+1 y se conecta al puerto 6000 del servidor para abrir el Canal de Datos.

Diagrama de Secuencia TCP

[Cliente] (Puerto 5000)   --- TCP SYN (Control) ---> [Servidor] (Puerto 21)
[Cliente] (Puerto 5000)   <-- TCP ACK (Control) ---  [Servidor] (Puerto 21)
[Cliente] (Puerto 5000)   --- Comando PASV ------->  [Servidor] (Puerto 21)
[Cliente] (Puerto 5000)   <-- Respuesta Puerto 6000- [Servidor] (Puerto 21)
[Cliente] (Puerto 5001)   --- TCP SYN (Datos) ---->  [Servidor] (Puerto 6000)
[Cliente] (Puerto 5001)   <-- TCP ACK (Datos) -----  [Servidor] (Puerto 6000)
🛠️ El Reto del Modo Pasivo: Como ahora los clientes se conectan a puertos aleatorios altos del servidor, el administrador del servidor debe configurar su Firewall para abrir un rango de puertos altos (ej. 50000 al 50100) y permitir que los clientes entren por ahí.

💡 Nota Arquitectónica Final (Seguridad): Tanto el FTP Activo como el Pasivo envían las contraseñas y los archivos en "texto plano". Es decir, cualquiera en la red puede leerlos. Hoy en día, los sistemas modernos utilizan SFTP (SSH File Transfer Protocol). SFTP no es FTP; funciona sobre una única conexión cifrada (Puerto 22) usando SSH, resolviendo tanto los problemas de Firewalls como los de seguridad de un solo golpe.
